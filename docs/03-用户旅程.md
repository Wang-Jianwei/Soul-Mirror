# 心镜 · 用户旅程 (User Journey)

> 配合 [02-功能体系.md](02-功能体系.md) 一起读  
> 哲学立基：阳明 · 事上练

---

## 总览

```
发现 → 首次打开 → 第一天 → 第一周 → 第一个月 → 第一年
 ↓        ↓         ↓        ↓         ↓          ↓
信任     入门      觉察     诊断     逍遥      本心档案
                       +物件    +立志
                       +一问    +节气
```

## 设计原则：70/30

**app 占用户生活的 30%，生活占 70%。**

app 是用户走进生活之前的**陪练**，不是用户生活的**替代**。
每一次 app 打开的结尾，都应该是"现在回去生活"。

## 五大关系线的 onboarding

| 关系线 | 何时引入 | 怎么引入 |
|---|---|---|
| **人与事** | 第 1 天 | 核心，觉察镜就是它 |
| **人与物** | 第 1 周 | app 安静地问："想选一个物件吗？" |
| **人与心** | 第 1 周 | app 列出几个预设问题，让用户选 1 个 |
| **人与时** | 第 1 天 | 被动，下次节气来时自然出现 |
| **人与人** | 第 1 个月 | 邀请制"指定 1 个同行者" |
| **人与无** | 第 1 天 | 静默日从一开始就存在 |

## 阶段详解

### 阶段 1 · 发现

**触点**：公众号 / 播客 / 朋友推荐

**用户心理**：
- "又一个自我提升 App？"
- "试试，但不抱希望"

**心镜的姿态**：
- 每一条内容都不带产品链接
- 只建立"这个团队在认真思考"的信任感
- 让对的人"找上门来"，而不是"被拉进来"

### 阶段 2 · 首次打开

**流程**（3 步内完成）：

1. 欢迎页：「此刻，你的心安吗？」
2. 选择状态（不安 / 平静 / 喜悦 / 迷茫）
3. 直接进入"今日"页

**关键说明**（首次）：
- 1 句话告诉用户"app 是道友，不是道场"
- "你大部分时间不需要我们"

**原则**：
- **无注册要求**（先体验，需要时再注册）
- **无功能介绍**（打开就是核心）
- **3 秒内**进入核心体验
- **不强制评估本性图谱**（V1 是可选）

### 阶段 3 · 第一天

**Aha moment**：

> 用户记下第一个念头  
> 输入："想买那门 2999 的课"  
> 心镜说：「这是本月第 3 次『知识焦虑』」  
> 然后问："今天你会做一件什么不一样的事吗？"  
> 用户写："今晚 11 点关手机，看窗外 10 分钟"  
> 心镜只说："好。"  
> **关 app。**  
> 今晚，用户真的关了 30 分钟。

**这一刻的克制**：
- 不弹"恭喜你完成了第一次记录！"
- 不发勋章
- 不引导"分享给朋友"
- 心镜**退一步**，让用户独自消化

### 阶段 4 · 第一周

**关键事件**：
- **物件选定**——app 安静地问"想选一个物件吗？不卖产品，你家里一定有什么能当这个角色"
- **一问选定**——app 列出几个预设问题，让用户选 1 个
- **第 1 个静默日**（周 X）出现——首页空白，只一行字："今天不需要心镜。"
- **第 1 次周观**（四谛 + 义利 + 五毒）

**仪式**：
- 周日晚自动生成"此心光明"小结（**不推送**）
- 用户主动打开
- 不显示"连续 N 周"等激励性指标

### 阶段 5 · 第一个月

**关键行为**：
- 邀请指定同行者
- 完成本性图谱评估
- 至少 1 次月度立志
- 物件日记积累 30 天，可导出

**同行者引入**：
- 不推送、不催促
- "想不想有个人一起走一段？"
- 用户指定后，app 提供极简的 check-in 通道

**留存机制（无为的留存）**：

| 行业惯例 | 心镜的做法 | 哲学依据 |
|---------|----------|---------|
| 推送通知 | 零推送 | 阳明 · 心只在主动时显现 |
| 红点 / 未读 | 无 | 老庄 · 无为 |
| 连续打卡奖励 | 无 | 孟子 · 义利之辨（不为奖励而活） |
| 周年活动 | 每年 12/31 自动生成档案 | 阳明 · 此心光明 |

持续使用的唯一理由：**内在的安宁**。

### 阶段 6 · 第一年

**关键事件**：
- **12/31 自动生成"此心光明"年度档案**
- **一问自答**（用户给未来的自己写一段）
- **物件日记导出**（这是用户独有的资产）
- **邀请用户换物件 / 换一问**（新一年）
- **不强制分享**

**目标**：
- 让用户感到"我变了"——不是心镜让他变了，是他自己
- 让用户成为"老用户"——不是因为积分，而是因为认同

## 一日样本（v1 想象）

```
06:30  起床，看到床头那枚铜钱（人与物）—— 没打开 app
07:15  出门前，看一眼那枚铜钱。今天的小事："中午给妈打个电话"
12:10  真的打了。心里记着。app 没出现
22:00  躺在床上，打开 app，写下"今日一行：✓ 给妈打了电话"
       + 念头一条："买那门课"
       + 关 app
23:50  app 静默，无推送。用户睡了

明天
06:30  铜钱还在床头。app 在等待被打开
```

**这就是"事上练"的样子。app 的工作是"等在生活之外"，不让生活被打扰。**

## 流失用户

**心镜允许用户离开。**

**心镜的态度**：
- 用户用得少 = 用户的本心不需要心镜
- 用户离开 = 用户已经在用自己的方式活
- 心镜的最高追求：用户不再需要心镜

**这与所有同行产品相反**——同行追求留存，心镜追求让用户能离开。

## 关键 Aha moment 排序

按重要性排：

1. **第一次"被看见"**（念头被识别为重复模式）
2. **第一次物件出现**（用户感受到物理锚点的重量）
3. **第一次静默日**（用户发现 app 不出现也是一种功能）
4. **第一次周观**（看见一周的本心轨迹）
5. **第一次决策 + 行**（V2 才会有，AI 时代的关键 aha）
6. **第一次年度档案**（长期承诺的回报）

## 反模式：心镜避免的引导

| 反模式 | 心镜的做法 |
|--------|----------|
| 新手教程（5 屏引导） | 打开即核心 |
| "完善个人资料" | 默认匿名 |
| "邀请好友得 Pro" | 无邀请奖励 |
| "你已坚持 7 天！" | 静默 |
| "升级 Pro 解锁更多" | 永远在主功能内，不卖 |
| "你错过了今天的题目" | 错过就错过 |
| "再写一行吧！" | 写完就关 |
| "X 个朋友也在用" | 不展示 |
