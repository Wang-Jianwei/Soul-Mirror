## 1. 产品定位

### 1.1 一句话描述
**一个用苏格拉底式提问帮你剥离情绪、看清本心的 AI 决策助手。**

### 1.2 核心价值
- **剥离**: 区分恐惧与真实危险
- **澄清**: 看见欲望背后的真正需要
- **决策**: 生成可回顾的决策卡片

### 1.3 使用场景
> 面临两难选择：接受高薪但 996 的 offer，还是留在现在的舒适区？  
> 打开良知决策，AI 问："如果完全不担心钱，你的本心会怎么选？"  
> 深入对话 5 轮后，发现恐惧被放大，本心早已明确。  
> 生成决策卡片，3 个月后回顾验证。

## 2. AI 架构设计

### 2.1 多模型策略

**设计原则**: 模型可插拔，根据场景和成本自动/手动切换

```
┌─────────────────────────────────────────┐
│           AI 路由层 (AI Router)          │
│         根据策略选择模型 + 失败降级       │
└─────────────────────────────────────────┘
                    │
    ┌───────────────┼───────────────┐
    │               │               │
    ▼               ▼               ▼
┌─────────┐   ┌─────────┐   ┌─────────┐
│ Claude  │   │  GPT-4  │   │ 本地模型 │
│  (主力) │   │ (备选)  │   │ (离线)  │
│ 云端API │   │ 云端API │   │ on-device│
└─────────┘   └─────────┘   └─────────┘
    │               │               │
    └───────────────┴───────────────┘
                    │
            ┌───────┴───────┐
            │   缓存层       │
            │ 常用响应本地缓存 │
            └───────────────┘
```

### 2.2 模型角色分工

| 模型 | 场景 | 优先级 | 成本 |
|------|------|--------|------|
| **Claude 3.5 Sonnet** | 复杂决策对话、深度追问 | 首选 | 中 |
| **GPT-4o mini** | 简单决策、快速响应 | 备选 | 低 |
| **GPT-4o** | Claude 不可用时降级 | 备选 | 高 |
| **本地 Llama 3 8B** | 完全离线、隐私敏感 | 兜底 | 免费 |

### 2.3 路由策略

**自动选择逻辑**:
```typescript
function selectModel(context: DecisionContext): AIModel {
  // 1. 检查网络状态
  if (!isOnline) return 'local-llama';
  
  // 2. 检查用户偏好
  if (userPreference) return userPreference;
  
  // 3. 根据复杂度选择
  if (context.complexity === 'high') return 'claude-sonnet';
  if (context.complexity === 'low') return 'gpt-4o-mini';
  
  // 4. 默认
  return 'claude-sonnet';
}
```

**失败降级链**:
```
Claude → GPT-4o → GPT-4o-mini → 本地模型 → 预设模板
```

### 2.4 成本控制策略

| 策略 | 说明 |
|------|------|
| **对话长度限制** | 最多 10 轮，防止无限对话 |
| **Token 预算** | 单次对话上限 4000 tokens |
| **响应缓存** | 相似问题命中缓存，不调用 API |
| **离线优先** | 本地模型处理简单分类/总结 |
| **用户选择** | Pro 用户可选模型，免费用户用低成本模型 |

### 2.5 成本估算

| 模型 | 每千 tokens | 单次对话成本 | 月活1万成本 |
|------|------------|-------------|------------|
| Claude 3.5 Sonnet | $3 / $15 | ~$0.05 | ~$500 |
| GPT-4o | $5 / $15 | ~$0.08 | ~$800 |
| GPT-4o mini | $0.15 / $0.6 | ~$0.003 | ~$30 |
| 本地 Llama | $0 | $0 | $0 |

**目标**: 人均月成本 <$0.1

#### 第二步：AI 苏格拉底式提问

**对话示例**:

> **用户**: 要不要接受这个 offer？钱多30%，但要996，搬家去上海。

> **AI**: 我们先剥离一下。这个决定里，有多少是恐惧？

> **用户**: 害怕现在的工作不稳定。

> **AI**: 如果完全不用担心钱和稳定，你的本心会怎么选？

> **用户**: 可能会选现在这个，有时间陪家人。

> **AI**: 那么，这个「害怕不稳定」，是真实的危险，还是被放大的恐惧？

> **用户**: 可能是被放大的，现在工作其实还行。

> **AI**: 那996和搬家，对你的本心来说，代价是什么？

...

**提问策略**:

| 阶段 | 目标 | 示例问题 |
|------|------|---------|
| **剥离** | 区分情绪与事实 | "这个决定里，有多少是恐惧？" |
| **假设** | 排除干扰因素 | "如果不担心钱，你会怎么选？" |
| **溯源** | 找到真实需求 | "你真正想要的是什么？" |
| **代价** | 评估机会成本 | "这个选择的代价是什么？" |
| **本心** | 澄清内在声音 | "你的本心在说什么？" |

**系统 Prompt 核心**:
```
你是心镜的「良知决策助手」，基于阳明心学和现代心理学，
用苏格拉底式提问帮助用户剥离情绪、看清本心。

原则：
1. 不替用户做决定，只帮用户看清
2. 每个问题都指向「本心」
3. 区分「恐惧」与「真实危险」
4. 区分「欲望」与「真正需要」
5. 最多10轮对话，最后生成决策卡片

语气：温和、坚定、不评判、不焦虑
```

### 4.2 本地模型方案

**选择**: llama.cpp + Llama 3 8B Instruct (Q4 量化)

| 指标 | 数值 |
|------|------|
| 模型大小 | ~4.5GB |
| 内存占用 | ~6GB |
| 推理速度 | 10-20 tokens/秒 (手机) |
| 能力 | 基础对话、简单分类 |

**使用场景**:
- 完全离线环境
- 隐私敏感决策
- 网络不稳定时降级

**限制**:
- 不支持复杂多轮对话
- 使用简化 Prompt
- 响应质量低于云端模型
