## 1. 产品定位

### 1.1 一句话描述
**一个帮你记录念头、看见模式的极简工具。**

### 1.2 核心价值
- **觉察**: 记录当下的念头
- **看见**: 回顾模式，发现重复
- **简单**: 无学习成本，3秒上手

### 1.3 目标用户
- 感到焦虑但不知为何的人
- 想自我觉察但坚持不下来的人
- 厌倦复杂效率工具的人

### 1.4 使用场景
> 晚上10点，躺在床上刷手机，突然想买一门课。  
> 打开心镜，记录："想买2999的AI课，害怕被淘汰"。  
> 看到提示：「这是本月第3次「知识焦虑」」。  
> 愣一下，放下手机，睡觉。

- 多行文本输入框
- 占位符："此刻，你在渴什么？"
- 最大 500 字
- 必填

**强度评分**
- 5级滑动条（1-5）
- 默认 3
- 视觉：低 → 高，颜色渐变

**类型选择**
- 6个预设类型（单选）
  - 知识焦虑（课程、书籍、信息）
  - 物质欲望（购物、消费）
  - 关系渴望（社交、认同、爱情）
  - 成就驱动（升职、赚钱、成功）
  - 逃避行为（游戏、短视频、拖延）
  - 其他
- 默认不选，用户手动选择
- MVP 不做 AI 自动分类

**记录后反馈**
- 轻提示：「已记录」
- 不跳转，留在页面可继续记录
- 或返回首页

## 3. 数据模型

### 3.1 SQLite 表结构

```sql
-- 念头记录表
CREATE TABLE thoughts (
  id TEXT PRIMARY KEY,           -- UUID
  content TEXT NOT NULL,         -- 内容
  intensity INTEGER DEFAULT 3,   -- 强度 1-5
  type TEXT,                     -- 类型
  created_at INTEGER NOT NULL    -- 时间戳
);

-- 每日一问回答表
CREATE TABLE daily_answers (
  id TEXT PRIMARY KEY,           -- UUID
  day_number INTEGER NOT NULL,   -- Day N
  question TEXT NOT NULL,        -- 问题内容
  answer TEXT,                   -- 回答内容
  created_at INTEGER NOT NULL    -- 时间戳
);

-- 心安状态记录表
CREATE TABLE mood_records (
  id TEXT PRIMARY KEY,           -- UUID
  mood TEXT NOT NULL,            -- 状态：anxious/calm/joy/confused
  created_at INTEGER NOT NULL    -- 时间戳
);
```

### 3.2 类型定义

```typescript
// 念头类型
type ThoughtType = 
  | 'knowledge_anxiety'  // 知识焦虑
  | 'material'           // 物质欲望
  | 'relationship'       // 关系渴望
  | 'achievement'        // 成就驱动
  | 'escape'             // 逃避行为
  | 'other';             // 其他

// 心安状态
type Mood = 'anxious' | 'calm' | 'joy' | 'confused';

// 念头记录
interface Thought {
  id: string;
  content: string;
  intensity: number;  // 1-5
  type?: ThoughtType;
  createdAt: number;
}

// 每日回答
interface DailyAnswer {
  id: string;
  dayNumber: number;
  question: string;
  answer: string;
  createdAt: number;
}

// 状态记录
interface MoodRecord {
  id: string;
  mood: Mood;
  createdAt: number;
}
```
