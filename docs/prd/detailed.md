## 1. 产品概述

### 1.1 产品定位

**Soul Mirror (心镜)** 是一个基于「不变人性」的个人觉醒产品，帮助用户在焦虑时代找到内心的平静。

### 1.2 核心价值主张

> **不是帮你做更多，而是帮你看清什么值得做。**

### 1.3 目标用户

| 维度 | 描述 |
|------|------|
| **年龄** | 25-40 岁 |
| **职业** | 知识工作者、创意从业者、中层管理者 |
| **收入** | 月收入 15K-50K |
| **城市** | 一二线城市 |
| **教育** | 本科及以上 |

### 1.4 用户心理画像

**小明，32岁，产品经理**
- 工作 5 年，年薪 40 万，但感到空虚
- 每天被各种效率工具绑架，反而更焦虑
- 买过 20+ 在线课程，完成率不到 10%
- 不知道自己真正想要什么
- 害怕 35 岁危机，但不知道往哪走

**小红，28岁，设计师**
- 工作 3 年，想转行但不知道转什么
- 被「自律」「成长」的概念压得喘不过气
- 社交媒体上看着别人成功，感到自卑
- 想要简单、真实的生活，但不知道怎么开始

## 2. 用户研究

### 2.1 痛点分析

| 痛点 | 严重程度 | 频率 | 现有解决方案 |
|------|---------|------|-------------|
| 被效率工具绑架 | ⭐⭐⭐⭐⭐ | 每天 | 放弃使用，但焦虑依旧 |
| 不知道自己想要什么 | ⭐⭐⭐⭐⭐ | 每周 | 做测试、咨询，但没用 |
| 决策时被情绪裹挟 | ⭐⭐⭐⭐ | 每月 | 问朋友、抛硬币 |
| 感到孤独迷茫 | ⭐⭐⭐⭐ | 每周 | 刷社交媒体，更焦虑 |

### 2.2 用户访谈关键发现

1. **"我不想再被『优化』了"**
   - 用户厌倦了效率工具带来的压力
   - 想要「被允许慢下来」的感觉

2. **"我知道自己的问题，但改不了"**
   - 自我认知足够，但缺乏行动支持
   - 需要「温柔的提醒」而非「严厉的监督」

3. **"我想看到真实的人，不是成功案例"**
   - 对「逆袭故事」免疫了
   - 想要「跟我一样普通的人怎么活」

## 3. 用户旅程

### 3.1 新用户旅程

```
发现 → 下载 → 首次打开 → 记录第一次 → 看到洞察 → 持续使用 → 分享
```

#### 阶段 1: 发现 (Awareness)

**触点**: 播客、公众号、朋友推荐

**用户心理**: 
- "又一个自我提升 App？"
- "试试，但不抱希望"

**设计目标**: 
- 传递「反效率」的差异化定位
- 建立信任感

#### 阶段 2: 首次打开 (Onboarding)

**流程**:
1. 欢迎页：「此刻，你的心安吗？」
2. 选择当前状态（不安/平静/喜悦/迷茫）
3. 简要说明：「心镜不是工具，是镜子」
4. 进入「今日」页面

**设计原则**:
- 无注册要求，先体验
- 无功能介绍，直接开始
- 3 秒内进入核心体验

#### 阶段 3: 第一次记录 (Aha Moment)

**场景**: 用户记录第一次「念头」

**理想体验**:
- 输入："想买那门 2999 的课"
- 系统提示：「这是本月第 3 次『知识焦虑』」
- 用户反应：「原来我是这样...」

#### 阶段 4: 持续使用 (Retention)

**关键行为**:
- 每周记录 3+ 次
- 完成本性评估
- 使用决策辅助

**留存策略**:
- 无推送，靠内在动力
- 每周「此心光明」小结
- 看到变化趋势

#### 阶段 5: 分享 (Advocacy)

**触发点**:
- 年度回顾生成
- 重大决策后
- 本性图谱完成

**分享形式**:
- 精美的卡片/海报
- 匿名故事投稿
- 邀请朋友（无奖励）

### 4.2 功能详细规格

#### 4.2.1 念头追踪 (Thought Tracker)

**功能 ID**: FT-001  
**优先级**: P0  
**状态**: 待开发

##### 用户故事

> 作为一个容易被念头带跑的人，
> 我想要快速记下当下的念头，
> 以便看清自己的心念模式。

##### 功能流程

```
触发 → 记录 → 分类 → 归档 → 洞察
```

**1. 触发**
- 用户主动打开 App
- 每日提醒（可选，默认关闭）
- 快捷方式（iOS 14+ Widget）

**2. 记录**

输入方式：
- 文字输入（默认）
- 语音输入（长按）
- 快捷标签（常用念头）

输入内容：
- 念头描述（必填）
- 强度评分（1-10，可选）
- 身体感受（可选）

**3. 分类**

自动分类（AI）：
- 知识焦虑（课程、书籍）
- 物质欲望（购物、消费）
- 关系渴望（社交、认同）
- 成就驱动（升职、赚钱）
- 逃避行为（游戏、短视频）

手动修正：
- 用户可修改分类
- 学习用户习惯

**4. 洞察**

个人模式：
- 本周念头次数
- 最常出现的类型
- 满足率统计
- 重复模式识别

对比洞察：
- 「你本月『知识焦虑』出现 5 次，平均满足率 20%」
- 「工作日晚 10 点后，『逃避行为』增加 300%」

##### 验收标准

- [ ] 3 秒内完成一次记录
- [ ] 语音识别准确率 > 90%
- [ ] 自动分类准确率 > 80%
- [ ] 支持离线记录
- [ ] 数据本地加密存储

#### 4.2.2 每日一问 (Daily Question)

**功能 ID**: FT-002  
**优先级**: P0  
**状态**: 待开发

##### 用户故事

> 作为一个想要自我觉察的用户，
> 我想要每天回答一个深度问题，
> 以便持续探索内心。

##### 功能流程

**问题库**: 365 个问题，按主题分类

| 主题 | 示例问题 |
|------|---------|
| 自我认知 | 「如果失去所有身份标签，你是谁？」 |
| 价值观 | 「你最不愿意妥协的是什么？」 |
| 关系 | 「你在讨好谁？」 |
| 恐惧 | 「你正在逃避面对什么？」 |
| 欲望 | 「你在用『忙碌』逃避什么？」 |

**回答方式**:
- 文字（支持 Markdown）
- 语音转文字
- 图片（可选）

**历史查看**:
- 按时间轴查看
- 按主题筛选
- 年度回顾整合

##### 验收标准

- [ ] 每日 0:00 更新问题
- [ ] 支持补答（过去 7 天）
- [ ] 回答自动保存草稿
- [ ] 支持导出为日记

#### 4.2.3 本性图谱 (Nature Profile)

**功能 ID**: FT-003  
**优先级**: P1  
**状态**: 待开发

##### 用户故事

> 作为一个迷茫的职场人，
> 我想要了解自己的本性倾向，
> 以便找到适合自己的生活方式。

##### 7 项本性评估

**评估方式**: 70 题问卷（每题 10 秒）

| 本性 | 描述 | 高分特征 | 低分特征 |
|------|------|---------|---------|
| **生存优先** | 对安全、稳定的需求 | 风险厌恶、储蓄倾向 | 冒险精神、随遇而安 |
| **繁衍本能** | 对后代、传承的关注 | 重视家庭、育儿投入 | 丁克倾向、自我优先 |
| **群居归属** | 对群体、认同的渴望 | 社交活跃、在意评价 | 独来独往、不在乎他人看法 |
| **趋乐避苦** | 对愉悦、舒适的追求 | 享受生活、及时行乐 | 延迟满足、苦行僧式 |
| **自利利他** | 自我与他人的平衡 | 互惠互利、边界清晰 | 过度付出或自私 |
| **好奇探索** | 对未知、新奇的兴趣 | 爱尝试、易厌倦 | 安于现状、抗拒变化 |
| **共通情绪** | 情绪敏感度 | 共情能力强、易受影响 | 情绪稳定、理性主导 |

##### 结果呈现

**雷达图**:
- 7 维度可视化
- 与人群平均对比
- 历史变化追踪

**解读报告**:
- 本性组合分析
- 适合的生活方式
- 潜在冲突预警
- 发展建议

**相似匹配**:
- 找到本性相似的用户
- 推荐他们的故事
- 建立弱连接

##### 验收标准

- [ ] 问卷完成时间 < 15 分钟
- [ ] 结果准确度 > 80%（用户自评）
- [ ] 报告生成时间 < 3 秒
- [ ] 支持重新评估

#### 4.2.4 良知决策 (Conscience Decision)

**功能 ID**: FT-004  
**优先级**: P1  
**状态**: 待开发

##### 用户故事

> 作为一个面临两难选择的人，
> 我想要剥离情绪看清本心，
> 以便做出不后悔的决定。

##### 决策流程

```
描述困境 → AI提问 → 深入探索 → 澄清本心 → 生成决策卡
```

**1. 描述困境**
- 标题（一句话概括）
- 背景描述
- 选项列举
- 时间压力

**2. AI 苏格拉底式提问**

示例对话：

> **用户**: 要不要接受这个 offer？钱多 30%，但要 996，搬家去上海。

> **AI**: 我们先剥离一下。这个决定里，有多少是恐惧？

> **用户**: 害怕现在的工作不稳定。

> **AI**: 如果完全不用担心钱和稳定，你的本心会怎么选？

> **用户**: 可能会选现在这个，有时间陪家人。

> **AI**: 那么，这个「害怕不稳定」，是真实的危险，还是被放大的恐惧？

**3. 澄清本心**

引导用户区分：
- 恐惧 vs 真实危险
- 欲望 vs 真正需要
- 社会期待 vs 自我意愿
- 短期舒适 vs 长期价值

**4. 生成决策卡**

包含：
- 困境描述
- 本心发现
- 决策建议
- 3 个月回顾提醒

##### 验收标准

- [ ] 对话自然流畅（图灵测试通过率 > 70%）
- [ ] 平均对话轮数 < 10
- [ ] 决策后满意度 > 75%
- [ ] 支持导出决策卡

## 5. 数据模型

### 5.1 核心实体

#### 用户 (User)

```typescript
interface User {
  id: string;                    // UUID
  email?: string;                // 可选
  created_at: Date;
  updated_at: Date;
  
  // 设置
  settings: {
    theme: 'dark' | 'light';
    notifications: boolean;
    reminder_time?: string;      // "21:00"
  };
  
  // 关联
  nature_profile?: NatureProfile;
}
```

#### 念头记录 (ThoughtRecord)

```typescript
interface ThoughtRecord {
  id: string;
  user_id: string;
  
  // 内容
  content: string;               // 用户输入
  intensity?: number;            // 1-10
  
  // 分类
  category: DesireCategory;      // 自动/手动分类
  category_confidence: number;   // AI 置信度
  
  // 元数据
  created_at: Date;
  fulfilled_at?: Date;           // 标记满足时间
  satisfaction?: number;         // 满足后评分 1-10
  
  // 身体感受（可选）
  body_sensation?: string;
}

type DesireCategory = 
  | 'knowledge_anxiety'    // 知识焦虑
  | 'material'             // 物质欲望
  | 'relationship'         // 关系渴望
  | 'achievement'          // 成就驱动
  | 'escape'               // 逃避行为
  | 'other';
```

#### 本性图谱 (NatureProfile)

```typescript
interface NatureProfile {
  id: string;
  user_id: string;
  
  // 7 项本性评分 (0-100)
  scores: {
    survival: number;        // 生存优先
    reproduction: number;    // 繁衍本能
    belonging: number;       // 群居归属
    pleasure: number;        // 趋乐避苦
    self_interest: number;   // 自利利他
    curiosity: number;       // 好奇探索
    emotion: number;         // 共通情绪
  };
  
  // 问卷答案
  answers: number[];         // 70 题答案
  
  created_at: Date;
  updated_at: Date;
}
```

#### 决策记录 (Decision)

```typescript
interface Decision {
  id: string;
  user_id: string;
  
  // 困境描述
  title: string;
  context: string;
  options: string[];
  
  // AI 对话
  conversation: Message[];
  
  // 结果
  conclusion?: string;
  chosen_option?: number;
  
  // 回顾
  created_at: Date;
  review_at: Date;           // 3 个月后
  review_notes?: string;
}

interface Message {
  role: 'user' | 'assistant';
  content: string;
  timestamp: Date;
}
```

#### 每日回答 (DailyAnswer)

```typescript
interface DailyAnswer {
  id: string;
  user_id: string;
  
  // 问题
  question_id: string;
  question_content: string;
  question_theme: string;
  
  // 回答
  content: string;
  media?: string[];          // 图片 URL
  
  created_at: Date;
}
```

## 6. 界面规格

### 6.1 设计原则

1. **深色优先** - 默认深色模式，护眼
2. **极简主义** - 无多余元素，专注内容
3. **呼吸感** - 大量留白，不压迫
4. **一致性** - 统一的设计语言

### 6.2 色彩系统

```css
/* 主色调 */
--color-primary: #E8D5B7;        /* 米白，温暖 */
--color-secondary: #8B7355;      /* 棕色，沉稳 */

/* 背景 */
--color-bg-primary: #1A1A1A;     /* 深黑 */
--color-bg-secondary: #2A2A2A;   /* 次黑 */
--color-bg-tertiary: #3A3A3A;    /* 卡片 */

/* 文字 */
--color-text-primary: #F5F5F5;   /* 主文字 */
--color-text-secondary: #A0A0A0; /* 次文字 */
--color-text-tertiary: #666666;  /* 辅助文字 */

/* 状态 */
--color-success: #7D9B76;        /* 平静 */
--color-warning: #D4A574;        /* 不安 */
--color-error: #B87070;          /* 焦虑 */
```

### 6.3 字体规范

- **标题**: Noto Serif SC（衬线，有温度）
- **正文**: Noto Sans SC（无衬线，清晰）
- **数字**: SF Mono（等宽，数据感）
