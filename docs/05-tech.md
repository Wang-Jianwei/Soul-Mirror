# 心镜 · 技术与路线图 (Tech & Roadmap)

> 配合 [02-features.md](02-features.md) 一起读

---

## 核心原则

1. **本地优先**：用户最私密的数据，留在设备上
2. **渐进披露**：先跑起来，再逐步加智能
3. **可解释**：AI 的每个建议，能说清楚为什么
4. **少即是好**：不为了"技术先进"而引入复杂度
5. **不依赖云端**：V1 完全离线可用

## 技术栈

### 客户端

| 技术 | 用途 | 备注 |
|------|------|------|
| React Native | 跨平台 | 一套代码双端 |
| TypeScript | 类型 | 强制 |
| Zustand | 状态 | 轻量 |
| React Navigation | 导航 | 标准 |
| expo-sqlite | 本地存储 | 加密（SQLCipher） |
| React Native Reanimated | 动效 | 仅必要微动效 |

### 后端（V1 不使用，V1.5+ 引入）

| 技术 | 用途 | 备注 |
|------|------|------|
| Supabase | BaaS | 认证 + 数据库 + 同步 |
| PostgreSQL | 数据库 | RLS 强制权限 |
| Edge Functions | 后端逻辑 | 轻量 |

### AI（V2 引入，V1 不使用）

| 技术 | 用途 | 备注 |
|------|------|------|
| 国内 LLM（MiniMax / 文心 / 通义） | 致良知对话 | 多模型路由 |
| 本地轻量模型 | 离线分类 | TensorFlow Lite |

## 关键决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 跨平台 | React Native | 一套代码双端，节省人力 |
| 本地 vs 云端 | V1 完全本地 | 与产品原则一致 |
| 推送 | 零 | 产品原则 |
| 账号 | V1 不需要 | 本地即可 |
| 同步 | V1.5+ 可选 | 慢引入 |
| AI | V2+ 引入 | V1 不依赖 |
| 区块链 / Web3 | 不用 | 与产品无关 |

## 安全

### 本地
- SQLite 加密（SQLCipher）
- 密钥存储 Keychain / Keystore
- 不写入系统剪贴板

### 云端（V1.5+）
- TLS 1.3
- 字段级加密
- RLS 强制

### 删除
- 一键清空（本地 + 云端）
- 不可恢复（不留 backup）

## 路线图

### Phase 1 · MVP（0-3 个月）

**目标**：验证 100 个种子用户愿意每日打开

**包含**：
- 觉察镜（此刻 / 良知一叩 / 念头）
- 简单周统计
- 一次性本性图谱

**技术**：
- React Native + SQLite
- 完全本地
- 无 AI、无云端

**里程碑**：
- 100 个种子用户
- 日活率 > 30%
- NPS > 50

### Phase 2 · 智能（3-9 个月）

**目标**：引入 AI，但不替代用户思考

**新增**：
- 致良知镜（AI 对话）
- 决策卡生成 + 回顾
- 模式识别（贪嗔痴慢疑自动标记）
- 多设备同步（云端）

**技术**：
- Supabase 后端
- 多模型 LLM 路由
- 本地轻量分类模型

**里程碑**：
- 1,000 付费用户
- 留存 > 40%
- 致良知对话满意度 > 75%

### Phase 3 · 陪伴（9-18 个月）

**目标**：从工具到长期陪伴

**新增**：
- 本心档案（季度 / 年度）
- 本性流变追踪
- 数据导出（PDF / 文本）
- 心镜·师 1v1 服务接入

**里程碑**：
- 10,000 付费用户
- 推出企业版

### Phase 4 · 生态（18-36 个月）

**目标**：成为"反焦虑"生活方式的基础设施

**探索**：
- 端侧大模型
- 端侧语义搜索（FTS5）
- 开放 API
- 极简硬件（"心镜"手机）
- 出版《此心光明》年度白皮书

## 风险与应对

| 风险 | 应对 |
|------|------|
| AI 成本过高 | 本地模型分担，优化 Prompt |
| 用户留存低 | 强化"年度回顾"的仪式感 |
| 竞品抄袭 | 速度不是壁垒，气质和信任是 |
| 技术债务 | 早期用成熟方案，不炫技 |
| 团队小 / 慢 | 慢增长不是问题，先验证再扩张 |
