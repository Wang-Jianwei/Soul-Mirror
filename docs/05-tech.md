# 心镜 · 技术与路线图 (Tech & Roadmap)

> 配合 [02-features.md](02-features.md) 一起读

---

## 核心原则

1. **本地优先**：用户最私密的数据，留在设备上
2. **渐进披露**：先跑起来，再逐步加智能
3. **可解释**：AI 的每个建议，能说清楚为什么
4. **少即是好**：不为了"技术先进"而引入复杂度
5. **不依赖云端**：V1 完全离线可用
6. **支持关系线**：技术服务于"五大关系"，不驱动功能堆叠

## 技术栈

### 客户端

| 技术 | 用途 | 备注 |
|------|------|------|
| React Native | 跨平台 | 一套代码双端 |
| TypeScript | 类型 | 强制 |
| Zustand | 状态 | 轻量 |
| React Navigation | 导航 | 标准 |
| expo-sqlite | 本地存储 | 加密（SQLCipher） |
| React Native Reanimated | 动效 | 仅必要微动效 |

### 后端（V1 不使用，V1.5+ 引入）

| 技术 | 用途 | 备注 |
|------|------|------|
| Supabase | BaaS | 认证 + 数据库 + 同步 |
| PostgreSQL | 数据库 | RLS 强制权限 |
| Edge Functions | 后端逻辑 | 轻量 |

### AI（V2 引入，V1 不使用）

| 技术 | 用途 | 备注 |
|------|------|------|
| 国内 LLM（MiniMax / 文心 / 通义） | 致良知对话 | 多模型路由 |
| 本地轻量模型 | 离线分类 | TensorFlow Lite |

## 关键决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 跨平台 | React Native | 一套代码双端，节省人力 |
| 本地 vs 云端 | V1 完全本地 | 与产品原则一致 |
| 推送 | 零 | 产品原则 |
| 账号 | V1 不需要 | 本地即可 |
| 同步 | V1.5+ 可选 | 慢引入 |
| AI | V2+ 引入 | V1 不依赖 |
| 物件 | V1 引入 | app 端"数字孪生" |
| 一问 | V1 引入 | 仅本地 |
| 节气 | V1 引入 | 仅本地、被动 |
| 静默日 | V1 引入 | 客户端判断 |
| 同行者 | V1.5 引入 | 必须有云端 + 账号 |
| 决策卡 + 行 | V2 引入 | 需 AI + 云端 |
| 区块链 / Web3 | 不用 | 与产品无关 |

## 安全

### 本地
- SQLite 加密（SQLCipher）
- 密钥存储 Keychain / Keystore
- 不写入系统剪贴板

### 云端（V1.5+）
- TLS 1.3
- 字段级加密
- RLS 强制

### 删除
- 一键清空（本地 + 云端）
- 不可恢复（不留 backup）
- 同行者看到的内容同步消失

## 路线图

### Phase 1 · MVP（0-3 个月）

**目标**：100 个种子用户每日打开，10 个愿意把"物件"坚持 1 个月

**包含**：
- **人与事**：此刻 / 念头 / 良知一叩 / 今日一行 / 简单周观 / 本周戒
- **人与物**：选定 + 物件日记
- **人与心**：一问（用户选 1 个问题）
- **人与时**：节气小邀请（被动）
- **人与无**：静默日（每周 1 天）
- **本性图谱**：一次性评估

**技术**：
- React Native + SQLite
- 完全本地
- 无 AI、无云端、无账号

**里程碑**：
- 100 个种子用户
- 日活率 > 30%
- 物件坚持率 > 10%（30 天）
- NPS > 50

---

### Phase 1.5 · 多人（3-5 个月）

**目标**：让"事上练"有一个同行者

**新增**：
- **同行者**：极简 1v1 check-in
  - 每日一行可"悄悄给同行者看一眼"
  - 没有评论、没有聊天、没有主页
- **季度互写**：每季双方各写一段
- **多设备同步**：云端同步
- **账号系统**：必需

**技术**：
- Supabase 后端
- 极简的 RLS（同行者只能看到一行 + 季度互写）

**里程碑**：
- 50% 活跃用户指定了同行者
- 季度互写完成率 > 30%

---

### Phase 2 · 智能（5-12 个月）

**目标**：引入 AI，但不替代用户思考

**新增**：
- **致良知镜**（AI 苏格拉底式对话）
- **决策卡 + 行**（生成 → 必填"何时做" → 3 个月回响）
- **7 天静默期**（决策卡生成后 app 主动消失）
- **预演模式**（为即将发生的事做准备）
- **模式识别**（贪嗔痴慢疑自动标记）
- **云端备份**（可选）

**技术**：
- 多模型 LLM 路由
- 本地轻量分类模型

**里程碑**：
- 1,000 付费用户
- 留存 > 40%
- 致良知对话满意度 > 75%

---

### Phase 3 · 陪伴（12-18 个月）

**目标**：从工具到长期陪伴

**新增**：
- **月度立志 / 改志**
- **本心档案**（季度 / 年度）
- **本性流变追踪**
- **数据导出**（PDF / 文本）
- **心镜·师** 1v1 服务接入（真人）
- **线下道友活动**（不靠 app 维持）

**里程碑**：
- 10,000 付费用户
- 推出企业版
- 心镜·师 服务试运营

---

### Phase 4 · 生态（18-36 个月）

**目标**：成为"反焦虑"生活方式的基础设施

**探索**：
- 端侧大模型
- 端侧语义搜索（FTS5）
- 开放 API（仅给"心镜·师"接入）
- 极简硬件（"心镜"手机）
- 出版《此心光明》年度白皮书

## 风险与应对

| 风险 | 应对 |
|------|------|
| AI 成本过高 | 本地模型分担，优化 Prompt |
| 用户留存低 | 强化"年度档案" + "物件日记" |
| 竞品抄袭 | 速度不是壁垒，气质和信任是 |
| 技术债务 | 早期用成熟方案，不炫技 |
| 团队小 / 慢 | 慢增长不是问题，先验证再扩张 |
| 关系线冷启动难 | 慢增长，承认"关系"不是流量 |
| 静默日被当成 bug | 首次打开就告知"静默是功能" |
