## 形状：道友不是道场

### 30/70 原则

**app 30%，生活 70%。**  
app 陪用户想清楚事，催用户走出去做。事发生在哪里，修行就在哪里。

### app 的四个角色

1. **轻触点**——每个关系线一个最小入口
2. **见证者**——记下用户回来说的话
3. **提醒者**——节气、物件、问题、一行
4. **退让者**——在重大事件后主动消失

## 五大关系线

### 一、人与事

**哲学立基**：阳明 · 事上练 / 知行合一

**模块**：

| 模块 | 内容 | 节奏 |
|------|------|------|
| **此刻心安** | 不安 / 平静 / 喜悦 / 迷茫 状态打卡 | 每日 |
| **念头一记** | 升起又灭去的念头 | 随时 |
| **良知一叩** | 今日，本心说了什么？ | 每日，365 题轮换 |
| **今日一行** | 1 句具体、时间明确的小事 | 每日（必填的尾） |
| **预演模式** | 为即将发生的事做准备 | 随时 |
| **见证模式** | 为已发生的事复盘 | 随时 |
| **决策卡 + 行** | AI 致良知对话 → 决策卡 → 行 → 3 个月回响 | 随时（V2） |
| **周观** | 四谛 + 义利之辨 + 五毒 | 每周 |
| **本周戒** | 1 件不做或 1 件要做 | 每周 |
| **7 天静默期** | 决策卡生成后 app 主动消失 | V2 |

**关键设计**：
- "**今日一行**"是必填的尾：每件念头/状态/叩之后必须 1 句"今天我会做 X"
- 行的具体：1 句、具体、时间明确（不是"对自己好一点"，是"今晚 11 点关手机，看窗外 10 分钟"）
- 决策卡生成后必须填"我会怎么做、何时做"
- 3 个月回响时先问"你做了吗"
- 没做不评判，下次再说

**对应学校**：阳明 · 事上练 / 知行合一

---

### 二、人与物

**哲学立基**：老庄 · 道法自然（外物即修行）、阳明 · 心外无物

**模块**：

- **选定**：用户自选 1 个物件（笔/石头/卡片/铜钱），app 不卖、不推荐、不联网商店
- **放置**：app 提示"把它放在你明天会看到的地方"（书桌/床头/玄关/出门必经）
- **触发**：app 不知道；但用户自己会知道。物件是 app 的延伸，**不是 app 的功能**
- **对话**：每天最后，可选写"今天它提醒了我什么"（1 句），形成"物件日记"

**关键设计**：
- 不上链、不 NFC、不物联网
- 物件在物理世界，app 是数字孪生
- 年度导出"物件日记"——这是用户独有的资产

**对应学校**：老庄 · 道法自然 / 阳明 · 心外无物

---

### 三、人与人

**哲学立基**：阳明 · 讲学 / 朋友

**模块**：

- **指定**：用户指定 1 个同行者（朋友/伴侣/师/咨询师），只能 1 个
- **极简异步**：每日一行可"悄悄给同行者看一眼"（一次按钮，无通知）
- **季度互写**：每季双方各写一段"你这季我看到的你"，150 字以内，互相读到
- **零扩展**：没有 feed、没有评论、没有点赞、没有主页

**关键设计**：
- 同行者是镜子的延伸，**不是社交**
- 用户可随时更换同行者
- 同行者必须有 app 账号才能用此功能
- 退订所有 = 同行者看到的内容也消失

**对应学校**：阳明 · 讲学 / 朋友

**V1 范围**：留 V1.5 引入

---

### 四、人与心

**哲学立基**：阳明 · 良知、孟子 · 四端

**模块**：

- **选定**：用户选 1 个问题（预设 or 自写），陪 1 年
  - 预设：「你今天对谁说了真话？」「你今天在逃避什么？」「你的本心在说什么？」
- **无处不在**：app 每次打开都显示这个问题
- **不追踪**：app 不记录"你思考了几次"
- **年度自答**：12/31 用户自写一次回答，给未来的自己（可存本地 / 打印 / 烧掉 / 埋掉）

**关键设计**：
- 问题不是功能，是用户与自己的关系
- app 提供**问题**，不提供**答案**

**对应学校**：阳明 · 良知

---

### 五、人与时

**哲学立基**：老庄 · 道法自然

**模块**：

- **节气仪式**：每个节气 1 个小邀请
  - 春分：种/养一株植物
  - 夏至：找一处荫凉独坐 30 分钟
  - 秋分：主动放下一样你握了很久的东西
  - 冬至：早睡一天
- **日出日落**：本地化的小提示
  - 日落前 30 分钟："今天你说过的话，有几句是真心的？"
- **不打卡**：做完就完了，不存档、不复盘

**关键设计**：
- 跟随自然节奏，不人为制造 deadline
- 节气不强制，但有邀请感
- 用户可以忽略，没有"你错过了 XX 节气"

**对应学校**：老庄 · 道法自然

---

### 六、人与无

**哲学立基**：老庄 · 无为 / 虚室生白

**模块**：

- **静默日**：每周一天首页空白，只一行字："今天不需要心镜。"
- **重大事件后**：致良知镜生成决策卡后，app 静默 7 天
- **年度禁言**：用户可选一年里的某一周（生日/忌日/任意）作为"心镜周"，app 只显示一句"此心光明"，其他全关
- **退订一切**：所有功能可一键关闭

**关键设计**：
- 静默是产品，不是 bug
- 留白是功能，不是缺失
- "虚室生白"——空了的房间才有光

**对应学校**：老庄 · 无为

## 时间节奏

| 节奏 | 触点 | 关联的关系线 |
|---|---|---|
| **每日** | 此刻 / 念头 / 良知一叩 / 今日一行 | 人与事 + 人与物 + 人与心 |
| **每周** | 周观 / 本周戒 / 静默日 | 人与事 + 人与无 |
| **随时** | 预演 / 致良知镜 / 决策卡 / 行 | 人与事 + 人与人 |
| **每月** | 月度立志 / 改志 / 月度复盘 | 人与事 + 人与心 |
| **季/年** | 季度互写 / 年度档案 / 年度立志 / 一问自答 | 人与人 + 人与事 + 人与心 |
| **静默** | 静默日 / 重大事件后 / 年度禁言 | 人与无 |
| **节气** | 24 个节气小邀请 | 人与时 |

**核心原则**：无推送，全部用户主动。**本心只在主动时显现。**

## 关键触点（app 30%）

app 只保留 5 个最小入口：

1. **今日** — 此刻心安 + 良知一叩 + 今日一行 + 一问（"今日"页）
2. **念头** — 念头历史 + 模式看见（"觉察"页）
3. **决策** — 预演 + 致良知镜 + 决策卡 + 行（"觉察"内）
4. **物件** — 选定 + 物件日记（"逍遥"内）
5. **心斋** — 静默 + 节气 + 一问（"逍遥"内）

底部 4 个 tab：今日 / 觉察 / 逍遥 / 我的（保留 v0.5 的形态）

## V1 明确不做

| 不做 | 理由 |
|------|------|
| 逍遥社区 | 社交容易变成比较 |
| AI 心理陪伴 | AI 是镜子，不是伴侣 |
| 公开排行榜 | 比较是焦躁的根 |
| 课程 / 内容流 | 心镜不输入，只整理 |
| 跨平台账号系统 | V1 完全本地 |
| 推送通知 | 用户主动，本心才显现 |
| 红点 / 未读数字 | 不被"未完成"绑架 |
| 任务清单 | "今日一行" 1 件，不是 TODO |
| 每日打卡 | 节气 / 物件 / 立志都不打卡 |
| 完美主义触发 | "你已经 3 天没记录了" |
| 跨设备同步 | V1.5 引入 |
| 公开排行 | 比较是焦躁的根 |

## MVP 范围（Phase 1）

**目标**：100 个种子用户每日打开，10 个愿意把"物件"坚持 1 个月

**包含**：
- **人与事**：此刻 / 念头 / 良知一叩 / 今日一行 / 简单周观
- **人与物**：选定 + 物件日记
- **人与心**：一问（用户选 1 个问题）
- **人与时**：节气小邀请（被动）
- **人与无**：静默日（每周 1 天）
- **本性图谱**：一次性评估

**不包含**（留 V1.5/2/3）：
- AI 致良知镜（Phase 2）
- 决策卡 + 行（Phase 2）
- 同行者（Phase 1.5）
- 月度立志 / 季度互写（Phase 3）
- 多设备同步（Phase 1.5）
- 云端备份（Phase 2）
