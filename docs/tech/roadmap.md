# 心镜 - 技术路径文档

## 核心原则

1. **本地优先**——用户最私密的数据，尽量留在设备上
2. **渐进披露**——先跑起来，再逐步加智能
3. **可解释**——AI 的每个建议，能说清楚为什么

## 阶段规划

### 第一阶段：MVP（0-3 个月）

**目标**：验证核心假设——有人愿意为「觉察」付费

**技术栈**：
- 前端：React Native（一套代码双端）
- 后端：Supabase（认证+数据库，省掉自建）
- AI：直接调用 Claude API，无自研模型

**核心功能**：
1. 渴爱追踪（纯本地表单，无 AI）
2. 每日一问（预设问题库，轮询）
3. 简单统计（本周记录了几次？）

**里程碑**：100 个种子用户，10 个付费转化

---

### 第二阶段：智能层（3-9 个月）

**目标**：AI 辅助，但不替代用户思考

**新增技术**：
- 本地轻量模型（TensorFlow Lite）：情绪分类、关键词提取
- 云端大模型：良知决策的引导对话
- 向量数据库（Pinecone）：用户历史记录的语义检索

**核心功能**：
1. **渴爱模式识别**
   - 输入："我又想买课了"
   - 本地模型识别：→ 「知识焦虑」模式
   - 调取历史：→ "上次买课后，你实际看了 12%"

2. **良知决策引导**
   - 不是给答案，是苏格拉底式提问
   - 云端 LLM 生成个性化追问链
   - 每次对话后，用户打标「有帮助/无帮助」

3. **本性图谱初版**
   - 7 项本性，每项 10 题问卷
   - 简单雷达图展示

**里程碑**：1,000 付费用户，留存率 > 40%

---

### 第三阶段：深度层（9-18 个月）

**目标**：从工具到「陪伴」，建立长期关系

**新增技术**：
- 本地语义搜索（SQLite + FTS5）：无需联网，检索自己 3 年前的记录
- 时序分析：识别用户周期的情绪波动
- 个性化内容生成：基于用户本性，生成「庄子式」短文

**核心功能**：
1. **年度「此心光明」回顾**
   - 自动生成：这一年，你的渴爱模式变化
   - 关键决策回顾：当时本心怎么说？现在回头看？
   - 导出为精美 PDF（可打印、可分享）

2. **逍遥社区**
   - 无点赞，无排名
   - 基于本性相似度匹配：「3 个跟你本性相似的人」
   - 内容审核：AI + 人工，确保无焦虑贩卖

3. **心镜·师接口**
   - 用户可授权数据给认证咨询师
   - 咨询师端：看用户历史记录，更高效对话

**里程碑**：10,000 付费用户，推出企业版

---

### 第四阶段：生态层（18-36 个月）

**目标**：成为「反焦虑」生活方式的基础设施

**技术探索**：
- 端侧大模型（手机本地跑 7B 模型）：完全离线，极致隐私
- 可穿戴集成：心率、睡眠数据纳入「觉察」
- 开放 API：第三方 App 可调用「本性图谱」

**商业模式扩展**：
- 硬件：「心镜」极简手机（仅通话 + 心镜 App）
- 出版：年度《此心光明》白皮书（用户匿名数据洞察）

## 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 本地 vs 云端 | 混合 | 敏感数据本地，AI 推理云端 |
| 自研 vs 第三方 AI | 第三方 | 早期用 Claude，后期考虑端侧 |
| 跨平台方案 | React Native | 省人力，性能够用 |
| 数据库 | SQLite + Supabase | 本地优先，云端同步 |
| 推送策略 | 零推送 | 产品原则，技术配合 |

## 风险与应对

| 风险 | 应对 |
|------|------|
| AI 成本过高 | 本地模型分担，优化 Prompt 长度 |
| 用户留存低 | 强化「年度回顾」的仪式感 |
| 竞品抄袭 | 速度不是壁垒，气质和信任是 |
| 技术债务 | 早期用成熟方案，不炫技 |

## 技术栈详细

### 前端
- React Native 0.72+
- TypeScript
- Zustand（状态管理）
- React Navigation

### 后端
- Supabase（PostgreSQL + Auth + Storage）
- Edge Functions（轻量后端逻辑）

### AI
- OpenAI / Claude API（云端）
- TensorFlow Lite（本地轻量模型）
- LangChain（AI 流程编排）

### 工具
- GitHub Actions（CI/CD）
- Sentry（错误监控）
- Amplitude（数据分析，可选）
