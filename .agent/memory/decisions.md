## ADR-001: 使用 React Native 作为跨平台方案

**状态**: ✅ 已接受  
**日期**: 2026-02-24  
**决策者**: Kimi Claw

### 背景

需要同时支持 iOS 和 Android，考虑开发效率和维护成本。

### 决策

使用 React Native 作为跨平台开发方案。

### 考虑过的选项

| 方案 | 优点 | 缺点 |
|------|------|------|
| React Native | 一套代码双端、生态成熟、热更新 | 性能略低于原生 |
| Flutter | 性能好、UI 一致 | Dart 学习成本、生态较小 |
| 原生开发 | 性能最好 | 两套代码、成本高 |

### 结论

选择 React Native，因为：
1. 团队已有 React 经验
2. 开发效率优先
3. 性能满足需求

## ADR-002: 本地优先的数据存储策略

**状态**: ✅ 已接受  
**日期**: 2026-02-24  
**决策者**: Kimi Claw

### 背景

用户数据高度敏感，需要保护隐私。

### 决策

采用本地优先策略：
- 核心数据存储在本地 SQLite
- 云端仅用于同步和备份（加密）
- 用户可随时导出或删除全部数据

### 理由

1. **隐私保护** - 最敏感的数据不出设备
2. **离线可用** - 无网络也能使用核心功能
3. **用户信任** - 数据自主权

## ADR-003: 使用 Supabase 作为后端服务

**状态**: ✅ 已接受  
**日期**: 2026-02-24  
**决策者**: Kimi Claw

### 背景

需要后端服务处理认证、同步、AI 推理。

### 决策

使用 Supabase：
- PostgreSQL 数据库
- 内置认证系统
- Edge Functions
- 实时订阅

### 理由

1. **开源** - 可自托管，避免 vendor lock-in
2. **功能齐全** - 一站式解决后端需求
3. **成本可控** - 免费额度充足

## ADR-004: AI 服务选择策略

**状态**: ✅ 已接受  
**日期**: 2026-02-24  
**决策者**: Kimi Claw

### 决策

采用混合策略：
- **云端大模型**: Claude API（复杂推理）
- **本地轻量模型**: TensorFlow Lite（简单分类）

### 理由

1. **隐私与效果平衡** - 敏感数据本地处理，复杂任务云端处理
2. **成本控制** - 本地模型减少 API 调用
3. **离线可用** - 基础功能无需联网
